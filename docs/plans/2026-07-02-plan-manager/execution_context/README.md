# Execution Context Artifacts

This directory contains the staged execution-context files used by the
orchestrated G -> T -> A implementation method.

These files are plan-support artifacts, not product runtime files. They exist so
each responsible agent receives only the common context plus the specific slice
for its assigned step. They must stay lean: no full sibling branches, no full
HRS/MRS unless the assigned step explicitly requires it and the escalation is
approved.

Generated bytecode caches, temporary dumps, and local scratch files are not
execution-context artifacts and must not be kept here.
