# Planning Standards Library

Purpose:
- Reusable planning, verification, and server-workflow standards for future projects
- Normative rule source for HRS/MRS/GS/TS/AS authoring and execution preparation

Use:
- Load only the standards required for the current mode and level
- Prefer references to these files over paraphrasing their rules in prompts
- Treat these standards as template-wide guidance, not project-specific artifacts

Primary files:
- `plan_standard_machine.yaml`
- `hrs_mrs_gs_consistency_verification_standard.yaml`
- `tactical_step_creation_standard.yaml`
- `atomic_step_creation_standard.yaml`
- `atomic_step_execution_standard.yaml`
- `TERMINAL_WORKFLOW.yaml`
- `editor_ca_workflow_prompt.yaml`
- `code_analysis_search_instructions.yaml`
- `code_analysis_fs_instructions.yaml`
- `code_analysis_universal_editing_instructions.yaml`

Supporting files:
